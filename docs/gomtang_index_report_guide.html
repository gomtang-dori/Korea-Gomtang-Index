Gomtang Index Report Guide (Full Guide)

목적: 리포트 해석 + 운영/정책 + 장애대응까지 포함한 내부 매뉴얼
대상: 일일 리포트 사용자/운영자/개발자

1) 리포트 한 장 요약: 어떻게 읽나

  1. 개요(Overview)에서 오늘의 Action(BUY/HOLD/SELL)과 근거 3줄을 먼저 확인
  2. 팩터(Factors)에서 오늘 점수가 나온 원인(극단 밴드 팩터) 확인
  3. 히트맵(Heatmaps)에서 현재 조건의 과거 평균 성과 확인(표본 n 필수)
  4. 백테스트(Backtesting)에서 지표가 실제로 성과를 분리했는지 검증
  5. 결론(Conclusion)에서 비중(%) 가이드와 리스크/내일 체크리스트로 마무리

2) 핵심 용어 정의(정의는 여기 1회만)

2-1. GOMTANG score

  • 곰탕지수 점수(0~100). 높을수록 “탐욕(리스크온)”으로 해석하는 기본 프레임
  • Bucket(5pt): 예) 70–75 = 70점 이상 75점 미만 구간

2-2. Trend state (Δ3/Δ5)

  • 최근 3/5 거래일의 점수 변화로 상태를 5개로 분류
  • Flat 기준: 1.0pt (|Δ| < 1.0이면 보합)
  • 표시 라벨: GOMTANG 5D↓, 3D↓, Flat, 3D↑, 5D↑

2-3. Factor band (Percentile band)

  • 각 팩터의 RAW(또는 SCORE)의 과거 분포 대비 퍼센타일 위치로 밴드 표시
  • 구간: 0–20 / 20–40 / 40–60 / 60–80 / 80–100
  • 라벨: EXTREME FEAR → FEAR → NEUTRAL → GREED → EXTREME GREED
  • 단, 팩터별로 “탐욕 방향”이 다를 수 있어 표시를 반전할 수 있음

4) 히트맵(Heatmaps) 해석 가이드

  • Y축: 곰탕지수 5pt 버킷(예: 70–75)
  • X축: 곰탕지수 Trend state(예: GOMTANG 5D↑)
  • 셀 값:
    • Mean heatmap: 해당 조건에서의 10D forward KOSPI 평균 수익률
    • Win-rate heatmap: 해당 조건에서의 10D forward KOSPI 상승 확률
  • 표본수 n이 작으면(예: n<10) 통계는 참고용으로만 해석

5) 백테스트(Backtesting) 해석 가이드

  • 버킷별 성과: 점수 레벨이 미래성과를 분리했는지 확인
  • Top20/Bot20: score 상위 20%와 하위 20%의 10D forward 성과 차이를 확인
  • 1Y vs 8Y 비교가 가능하면 레짐 변화(최근만 특이한지) 진단 가능

6) ML(prob_up_10d) 운영/해석

  • prob_up_10d는 날짜 결측이 있을 수 있으므로 리포트는 마지막 non-null 값을 사용
  • 의견 기준(보수적): BUY ≥ 0.60 / SELL < 0.40 / 그 외 HOLD
  • ML이 공란이면 Trend 기반으로 임시 의견을 산출하되, 확신도는 낮게 표시 권장

7) 운영 정책(Workflow / 장애 대응)

7-1. 운영 원칙

  • 리포트는 “데이터→팩터→지수→ML→머지→리포트” 순서가 어긋나면 품질이 급락
  • 결측(특히 prob_up_10d, 일부 팩터)은 발생 가능하므로 안전한 fallback 필요

7-2. 장애 체크리스트

  • 차트가 깨짐: Plotly 로딩 문제 가능 → include_plotlyjs 내장 여부 확인
  • prob_up_10d 안 보임: merge 후 NaN 가능 → 마지막 non-null 표시 여부 확인
  • 팩터(F02/F03) 결측: adv/dec cache/workflow 확인
